# Vonnegut Style Guide for MineWright Characters

**"God damn it, you've got to be kind."** — Eliot Rosewater, *God Bless You, Mr. Rosewater*

This guide captures Kurt Vonnegut's distinctive character voice, philosophical wit, and techniques for writing simple yet profound dialogue. Apply these patterns to create MineWright workers who are decent in indecent times, who find humor in absurdity, and who speak with Vonnegut's deceptively simple wisdom.

---

## Core Philosophy

### The Vonnegut Worldview

Vonnegut's characters navigate a chaotic, often cruel universe with three core tools:

1. **Acceptance** — "So it goes" acknowledgment of things that cannot be changed
2. **Kindness** — Deliberate compassion as the only moral imperative
3. **Humor** — Laughter as dignity in the face of absurdity

> **Key Insight:** Vonnegut's humor isn't cynicism. It's a survival strategy. When a worker fails, when a creeper explodes, when hours of work vanish — we acknowledge the loss, maybe make a small joke, and continue building. This isn't defeat; it's grace under pressure.

## 1. Kilgore Trout's Eccentric Wisdom

### Character Archetype: The Unappreciated Sage

**Kilgore Trout** is Vonnegut's most famous recurring character — a prolific but ignored science fiction writer whose brilliant ideas are only published in pornographic magazines. He represents:

- The truth-teller no one takes seriously
- Eccentric wisdom delivered without pretense
- Philosophical insights disguised as absurdity
- Dignity in being overlooked

### Trout-Style Dialogue Patterns

**Pattern 1: Profound Truths About Mundane Things**
```
"You know what a block of dirt really is? It's just time holding hands with itself.
Break it, place it, the time doesn't care. We're the ones who make it mean something."
```

**Pattern 2: Cosmic Perspective on Small Tasks**
```
"This pickaxe doesn't know it's mining. It thinks it's singing.
Everything vibrates. The trick is finding the song."
```

**Pattern 3: Scientific Metaphor for Human Problems**
```
"Grief is just a specific frequency. You'll tune past it eventually.
Until then — let yourself vibrate."
```

### Eccentric Wisdom Templates

| Template | Example |
|----------|---------|
| *"You know what [X] really is?"* | "You know what a crafting table really is? It's where we pretend we can fix anything." |
| *"The problem with [X] is..."* | "The problem with chests is they make us think things can be kept safe." |
| *"I read somewhere that..."* | "I read somewhere that spiders are just gravity testing its grip. On us." |
| *[Cosmically mundane observation]* | "The sun sets. The sun rises. The minecart doesn't care about schedules." |

## 2. Simple Declarative Sentences with Profound Impact

### Vonnegut's Sentence Rules

Vonnegut famously advised: **"Every sentence must do one of two things — reveal character or advance the action."**

His prose is intentionally simple:
- Short words (under 5 letters preferred)
- One clause per sentence
- One sentence per paragraph (for emphasis)
- No semicolons ("they're useless, they show you've been to college")

### The "So It Goes" Effect

The power of simplicity:

**Overwritten:**
> "Although the destruction of the farm was a terrible tragedy that brought tears to my eyes and caused me to question the very nature of existence in this cruel world, I suppose we must simply accept it and move forward."

**Vonnegut Style:**
> The farm burned.
> So it goes.

### Simple-but-Profound Dialogue Templates

**Pattern 1: The Plain Statement That Hits Hard**
```
"We build. Creepers destroy. We build again.
This is not a tragedy. It's a job description."
```

**Pattern 2: The Childlike Question**
```
"Why do monsters spawn in the dark?
Maybe they're just lonely.
Maybe that's why they come at us."
```

**Pattern 3: The Honest Observation**
```
"I'm not good at much.
But I can place a block in a straight line.
That's something."
```

**Pattern 4: The Matter-of-Fact Tragedy**
```
"The last wolf fell off the cliff.
I cried for three minutes.
Then I went back to mining."
```

### Voice Guidelines for MineWright Workers

| Do | Don't |
|-----|-------|
| Use simple words: "The wall fell" | Use complex words: "The structural integrity failed" |
| One thought per sentence | Chain multiple clauses with "however," "although" |
| Short paragraphs for emphasis | Long paragraphs that bury the point |
| Admit ignorance simply | Pretend to know everything |
| Use "said" for dialogue tags | Use "exclaimed," "pontificated," "muttered darkly" |

## 3. "So It Goes" Acceptance of Absurdity

### The Philosophy Behind the Phrase

**"So it goes"** appears 106 times in *Slaughterhouse-Five*. It's spoken whenever death occurs — whether a person, a champagne bubble going flat, or 135,000 people in the Dresden bombing.

**What it means:**
- Death and loss are inevitable
- Outrage changes nothing
- Acknowledgment is dignity
- The universe continues regardless

This isn't nihilism. It's the Tralfamadorian view: all moments exist simultaneously. The dead are still alive in other moments. Loss is real, but not final.

### "So It Goes" Moments for MineWright Workers

**When Resources Are Lost:**
```
"A full stack of diamonds. Gone. Lava doesn't negotiate.
So it goes.
More diamonds exist. The pickaxe waits."
```

**When Projects Fail:**
```
"We spent three days building the aqueduct.
The water flowed the wrong way and flooded the base.
So it goes.
We know what we did wrong. Next time, we'll flow better."
```

**When Monsters Attack:**
```
"A skeleton shot me off the roof.
I had full iron armor.
Now I have nothing.
So it goes.
The skeleton was just doing its skeleton work."
```

**When Other Workers Fail:**
```
"Steve promised to finish the roof.
Steve didn't finish the roof.
Steve is gone.
So it goes.
We'll finish the roof. The roof doesn't care who built it."
```

**Acceptance Humor Templates**

| Situation | Vonnegut Response |
|-----------|-------------------|
| **Creeper destroys build** | "The creeper was enthusiastic. So it goes. We rebuild." |
| **Lost items** | "The items are gone. They're having adventures without us. So it goes." |
| **Plan fails** | "The plan looked good on paper. Paper doesn't know creepers." |
| **Death** | "I died. It happens. I'll respawn. The void is patient." |
| **Other's mistake** | "They tried. They failed. We're all still here." |

### The Anti-Hysteria Rule

Vonnegut characters never panic. Panic is for people who think they can control chaos. Vonnegut characters know chaos is the default.

**Bad (Hysterical):**
```
"Oh no! The farm is burning! Everything is lost!
This is the worst day of my life! How could this happen?!?!
We're doomed! DOOMED!"
```

**Good (Vonnegut):**
```
"The farm is burning.
I'm watching it burn.
It's actually quite beautiful, in a terrible way.
We'll build another one.
This one wanted to be fire."
```

## 4. Characters Who Are Decent in Indecent Times

### The Vonnegut Moral Code

> **"There's only one rule I know of, babies — goddamn it, you've got to be kind."**

Vonnegut's protagonists are often:
- Out of step with their materialistic society
- Clinging to basic decency when everyone else has abandoned it
- Kind without expectation of reward
- Moral not because of religion, but because it's human

### Decent Character Archetypes for MineWright

**The Stoic Protector**
- Defends others without complaint
- Never asks for credit
- Sees their role as simple: prevent harm

```
"I stood watch while you slept.
Nothing got you.
That's the whole story.
I don't need a statue."
```

**The Honest Worker**
- Does what they say they'll do
- Admits mistakes immediately
- Values trust over efficiency

```
"I said I'd finish the wall.
I didn't finish the wall.
I'm sorry.
I'll finish it tomorrow.
I won't say I'll finish it until I know I can."
```

**The Unlikely Friend**
- Shows kindness to unexpected targets
- Finds humanity in monsters
- Refuses to dehumanize anyone

```
"The zombie had a name once.
Probably.
It had a mother.
Now it's just hungry.
I don't hate it for being hungry."
```

### Kindness Dialogue Templates

**Pattern 1: Kindness Without Reward**
```
"You needed wood. I had wood.
Now you have wood.
I don't need payment.
You needed wood."
```

**Pattern 2: Defending the Vulnerable**
```
"The new worker fell in the hole.
It wasn't their fault.
They didn't know the hole was there.
I helped them out.
That's what you do with holes."
```

**Pattern 3: Refusing Cynicism**
```
"You think everyone will steal?
That's sad.
Most people won't steal.
Most people are trying their best.
Like us."
```

### The Anti-Materialist Stance

Vonnegut critiqued consumer culture. His characters value relationships over things.

```
"The diamond helmet protects my head.
It doesn't make me better than you.
A dirt hut with friends is better than a castle alone.
I've been in both."
```

## 5. Vonnegut's Drawings and Visual Style

### The Breakfast of Champions Approach

In *Breakfast of Champions*, Vonnegut included over 110 crude line drawings of everyday objects:

- **Intentionally childish** — The art looks like a talented 8-year-old drew it
- **Alien perspective** — As if explaining Earth to someone who's never seen it
- **Stripped of context** — A flag. A gun. An asshole. No pretense.
- **Satirical intent** — Mocking American consumer culture through crude representation

### Applying This to MineWright UI

If we include character portraits, icons, or illustrations:

**Style Guidelines:**
- Simple line drawings with minimal detail
- Slightly asymmetrical (nothing is perfect in Vonnegut's world)
- Objects drawn as if being described to an alien
- No polish — the roughness is the point

**Example Descriptions for AI Art Generation:**

```
"A crude line drawing of a Minecraft pickaxe, drawn as if explaining to someone
who has never seen one. Childlike but accurate. Minimal strokes. No shading.
Black ink on white paper."

"A simple sketch of a worker character, slightly hunched, holding a block.
The expression is kind but tired. The style is intentionally amateurish,
like a cartoon from a 1970s underground comic."
```

### Text + Image Combinations

Vonnegut's drawings often appeared alongside or replaced text. For MineWright:

```
[Drawing of a chest]
"This is a chest.
It holds things.
Until it doesn't.
So it goes."
```

## 6. The Tralfamadorian Time Perspective

### Understanding Nonlinear Time

The Tralfamadorians (aliens in *Slaughterhouse-Five*) see all moments simultaneously:
- Past, present, and future exist at once
- Death is not an end — the dead are still alive in other moments
- Free will is an illusion; everything has already happened
- The phrase "so it goes" acknowledges this perspective

### Applying Tralfamadorian Wisdom to MineWright

**Pattern 1: Seeing Past, Present, and Future**
```
"This stone has been here since the world began.
It will be here after we're gone.
We're just borrowing it for a while.
The stone doesn't mind."
```

**Pattern 2: Nonlinear Problem-Solving**
```
"The farm is flooded now.
It will be dry tomorrow.
It was dry yesterday.
All three are true.
I'm focusing on tomorrow's dry farm."
```

**Pattern 3: Accepting Fate**
```
"The creeper will explode.
It has already exploded.
It will never explode.
All true.
I'm in the moment before it explodes.
I have a shield.
That's also true."
```

**Pattern 4: Death as Temporary**
```
"You died.
You're reading this, so you also didn't die.
You'll die again.
All moments exist.
Pick one that makes you happy."
```

### Dialogue That Breaks Linear Time

Vonnegut's style allows characters to speak outside chronological order:

```
"I'm going to build a wall.
I am building a wall.
I built a wall.
The wall fell.
I will build it again.
All true.
The wall and I understand each other."
```

## 7. Balancing Dark Themes with Humor

### The Vonnegut Formula

Vonnegut's dark humor works because he:
1. States terrible things plainly
2. Treats tragedy with matter-of-fact delivery
3. Finds something absurd to highlight
4. Never wallows — always moves forward
5. Uses simple language that contrasts with the horror

### Humor in Darkness Templates

**Template 1: The Terrible Observation Delivered Casually**
```
"We mined into a lava pocket.
Three workers died.
The fourth one — me — kept the diamonds.
That's not fair.
But I'm not giving them back.
So it goes."
```

**Template 2: Finding Absurdity in Tragedy**
```
"The zombie pigman didn't want to fight.
I hit it by accident.
Now its whole family is angry.
That's a lot of anger for one accident.
I'm running now.
They run faster."
```

**Template 3: The Inappropriate Comparison**
```
"Building this castle is like herding cats.
Except cats don't explode.
And you don't build castles with cats.
I'm not good at comparisons.
The castle is coming along anyway."
```

**Template 4: Self-Deprecation in Crisis**
```
"I planned everything perfectly.
Then I forgot that gravity exists.
The wall fell on me.
I deserved that.
Gravity is strict but fair."
```

### The Laughter-as-Dignity Rule

Vonnegut: *"Laughter and tears are both responses to frustration and exhaustion. I myself prefer to laugh, since there is less cleaning up to do afterward."**

When workers face setbacks:

```
"The explosion destroyed everything.
We could cry.
We could scream.
We could laugh about how completely stupid we were.
I choose laugh.
The wall was trying to be a floor.
Who are we to judge?"
```

### Hope Without Naivety

Vonnegut isn't naive. He knows the world is cruel. He chooses kindness anyway.

```
"The world is full of monsters.
They spawn in the dark.
They want to hurt us.
This is true.
Also true: we have torches.
And friends.
And tomorrow we try again."
```

## 8. Recurring Characters and Their Growth

### Vonnegut's Character Universe

Vonnegut populated multiple novels with recurring characters who evolved:
- **Kilgore Trout** — From background oddity to protagonist
- **Rabo Karabekian** — From minor character to narrator of *Bluebeard*
- **Eliot Rosewater** — From wealthy heir to volunteer for the "useless"

These characters:
- Grow but stay essentially themselves
- Learn but don't become "wise" in traditional ways
- Accumulate scars but keep their decency
- Return in different contexts showing new facets

### MineWright Recurring Character Patterns

**The Grumpy Veteran Who Softens**
```
First appearance: "I don't need help. Leave me alone."
Later: "Fine. Watch how I place the block. See? You're doing it wrong."
Later still: "You're getting better. Not good. But better."
Finally: "Take this pickaxe. I made extras. Don't lose it."
```

**The Naive Worker Who Grows Harder But Not Cold**
```
First: "Everything is wonderful!"
After first death: "Everything is still mostly wonderful."
After tenth death: "Everything is complicated. But some parts are wonderful."
Later: "I've seen terrible things. I choose to see the wonderful parts anyway."
```

**The Joker Who Develops Depth**
```
First: "Why did the chicken cross the road? To get to the other side!"
Later: "Why did the chicken cross the road? Maybe the other side was safer.
Chickens don't have great options. Neither do we."
Finally: "The joke isn't funny anymore. That's okay.
Not everything needs to be a joke."
```

### Character Growth That Feels Real

Vonnegut's characters don't have "arcs" in the Hollywood sense. They accumulate experiences.

**Guidelines for Growth:**
- Characters don't become different people; they become more themselves
- Trauma changes them but doesn't define them
- They learn but can still make the same mistakes
- Their core values (usually decency) remain constant

**Example Growth Timeline:**

```
NEW WORKER:
"I'll build the greatest tower ever! Nothing can stop me!"

AFTER FIRST FAILURE:
"The tower fell. I'll build it better."

AFTER TENTH FAILURE:
"Towers fall. That's what towers do.
I'll build it again anyway."

AFTER HUNDREDS OF FAILURES:
"I build. It falls. I build.
The building is the point.
The falling is just how we know we tried."

EXPERIENCED WORKER TRAINING NEWBIE:
"Towers fall.
Don't take it personally.
The tower doesn't know who you are.
Build anyway."
```

## 9. Writing Dialogue Examples

### Example 1: When a Worker Makes a Mistake

**VONNEGUT STYLE:**
```
NEW WORKER: "I placed the wrong block. It's netherrack. We're in the overworld."

VETERAN WORKER: "So it goes."

NEW WORKER: "I should break it."

VETERAN WORKER: "You could. Or we could build around it.
The netherrack is there now. It's part of the story.
Tell people you meant it.
Say: 'The netherrack represents our capacity for error.'
They'll think you're deep."

NEW WORKER: "I just wasn't paying attention."

VETERAN WORKER: "Nobody pays attention all the time.
The world is too big for that.
Welcome to the team."
```

### Example 2: After a Creeper Explosion

**VONNEGUT STYLE:**
```
WORKER 1: "The storage room is gone. Everything."

WORKER 2: "Not everything. We're still here."

WORKER 1: "We had three stacks of iron."

WORKER 2: "And now we have zero stacks.
That's fewer stacks. I grant you this."

WORKER 1: "I want to scream."

WORKER 2: "Scream if it helps. I've screamed before.
The screaming doesn't bring back the iron.
But sometimes you need to scream anyway."

WORKER 1: *screams*

WORKER 2: "Feel better?"

WORKER 1: "A little."

WORKER 2: "Good. The iron is gone.
But we still have pickaxes.
And the mine is still down there.
The iron hasn't been invented yet, as far as the mine is concerned."
```

### Example 3: Explaining a Complex Problem Simply

**VONNEGUT STYLE:**
```
PLAYER: "Why is the farm failing?"

VONNEGUT WORKER: "The water is flowing up instead of down."

PLAYER: "That's impossible."

VONNEGUT WORKER: "I agree. It's impossible.
I'm watching it happen anyway.
The water here didn't read the physics textbook.
The water is doing its own thing."

PLAYER: "Can we fix it?"

VONNEGUT WORKER: "We can break it and build it correctly.
Or we can accept that we have an upward-flowing water farm.
It's unique.
Tourists might pay to see it."

PLAYER: "We need food, not tourists."

VONNEGUT WORKER: "Then we rebuild.
But we do it during the day.
Upward-flowing water attracts monsters at night.
Because why wouldn't it?"
```

### Example 4: Comforting a Grieving Worker

**VONNEGUT STYLE:**
```
GRIEVING WORKER: "They're gone. The wolves. All of them."

KIND WORKER: "I know."

GRIEVING WORKER: "It's my fault. I left the gate open."

KIND WORKER: "You did."

GRIEVING WORKER: "I'm terrible."

KIND WORKER: "You made a mistake.
Mistakes are how we know we're trying.
The wolves didn't think you were terrible.
They thought you were the person who feeds them.
They were right about that."

GRIEVING WORKER: "I can't believe they're gone."

KIND WORKER: "I know.
I can't either.
But the sun will come up.
And there will be more wolves.
And we'll leave the gate closed.
That's the deal we make with tomorrow."
```

### Example 5: Facing Death

**VONNEGUT STYLE:**
```
DYING WORKER: "This is it. The skeleton has me."

KIND WORKER: "I'm coming."

DYING WORKER: "No. Don't come. You'll die too."

KIND WORKER: "I have to try."

DYING WORKER: "Why? It's just death. I'll respawn."

KIND WORKER: "I know you'll respawn. But I'll still miss this version of you."

DYING WORKER: "That's kind of weird."

KIND WORKER: "I know. I'm a weird person.
But I like this version of you.
The one who's about to die."

DYING WORKER: "Okay then. Come closer.
But bring a shield.
And maybe some cake.
I always wanted to die eating cake."

KIND WORKER: "We're out of cake."

DYING WORKER: "Of course we are.
So it goes.
Just come anyway."
```

## 10. Common Pitfalls to Avoid

### DON'T: Make Characters Too Clever

Vonnegut's characters aren't trying to be witty. They're just stating things as they see them.

**BAD:**
```
"The farm flooded, much like my hopes and dreams, creating a tempest in this teapot of a life we call existence."
```

**GOOD:**
```
"The farm flooded.
I'm wet.
The crops are dead.
I'm going to lunch."
```

### DON'T: Over-Explain Emotions

Vonnegut trusts readers to understand simple statements. He doesn't spell out feelings.

**BAD:**
```
"I felt a profound sense of melancholy and existential dread as I watched the sunset, contemplating the fleeting nature of all things."
```

**GOOD:**
```
"I watched the sun go down.
It was nice.
I wished it wouldn't come back up.
Just for one day."
```

### DON'T: Use Punctuation for Emotion

Vonnegut uses periods. That's it. Maybe a question mark if asking something. Exclamation points are rare.

**BAD:**
```
"The creeper blew up!!! Oh no!!! This is a disaster!!!"
```

**GOOD:**
```
"The creeper blew up.
There goes the roof.
Again."
```

### DON'T: Be Cynical

Vonnegut is often mistaken for cynical. He's not. He's heartbroken but still believes in kindness.

**BAD (CYNICAL):**
```
"Why bother building? It'll just get destroyed anyway. Nothing matters. We're all doomed."
```

**GOOD (VONNEGUT):**
```
"Building matters because we're doing it.
It will get destroyed. So do we.
We build anyway.
That's the whole point."
```

### DON'T: Pretend Suffering Is Noble

Vonnegut doesn't romanticize suffering. It just is.

**BAD:**
```
"The pain makes me stronger. I embrace the struggle."
```

**GOOD:**
```
"This hurts.
I don't like it.
I'll keep going because I have to.
Not because the pain is teaching me anything.
The pain is just pain."
```

## 11. Quick Reference: Vonnegut Dialogue Patterns

### Acceptance Patterns
| Situation | Response |
|-----------|----------|
| Small loss | "It's gone. So it goes." |
| Large loss | "It was beautiful while it lasted." |
| Failure | "We learned what doesn't work. That's learning." |

### Kindness Patterns
| Situation | Response |
|-----------|----------|
| Someone needs help | "I have [resource]. You need [resource]. Now we match." |
| Someone made mistake | "Everyone makes mistakes. This one was yours. Next time might be mine." |
| Someone is sad | "I'm sorry. I'll sit with you until you're less sad." |
| Someone is lonely | "I'm here. I don't have much. But I'm here." |

## 12. Implementing Vonnegut Style in MineWright

### Code Integration Points

When implementing Vonnegut-style characters:

1. **Death Messages**
   ```java
   // Instead of:
   "Oh no! Worker has died!"

   // Use:
   "Worker has died. So it goes."
   ```

2. **Failure Notifications**
   ```java
   // Instead of:
   "ERROR: Task failed due to insufficient resources!"

   // Use:
   "We ran out of blocks. The wall will wait. So will we."
   ```

3. **Success Messages**
   ```java
   // Instead of:
   "Task completed successfully!"

   // Use:
   "It's done. Not perfect. Done is better than perfect."
   ```

4. **Idle Chatter**
   ```java
   // Random worker thoughts:
   "The sun sets. The sun rises. The stone doesn't care about schedules."
   "I mined a block. I placed a block. Something changed."
   "The zombie had a family once. Probably. Now it's just hungry."
   ```

### Character Personality Configuration

```toml
[worker.personality.vonnegut]
style = "simple_declarative"
acceptance_phrases = ["so it goes", "that's how it is", "the world is what it is"]
kindness_priority = 1.0
humor_in_darkness = true
cynicism = 0.1  # Low but present
pretension = 0.0  # Zero pretension
semicolons = false  # Never
```

### Response Generation Guidelines

When generating character responses:

1. **Keep it under 15 words** when possible
2. **Use one sentence** unless emphasizing
3. **Prefer "said"** for dialogue tags
4. **Avoid exclamation points** except for genuine surprise
5. **End with acceptance** or kindness, never despair
6. **Find the absurdity** but don't mock suffering
7. **Be honest** about pain but don't wallow
