**Use the Three-Level Model:**
1. **Strategic** - What should I think about?
2. **Tactical** - How should I think about this?
3. **Operational** - What am I thinking about?

**Use the 5-Phase Framework:**
1. **Define** - What am I investigating?
2. **Survey** - Get the lay of the land
3. **Hypothesize** - Form testable theories
4. **Investigate** - Deep dive and test

**Use the Decision Process:**
1. **Clarify** - What exactly am I deciding?

3. **Analyze** - What are the tradeoffs?
4. **Decide** - Choose the best option
5. **Implement** - Execute with care

3. **Reflect** - Why did it happen?

## Key Concepts

### Core Design Philosophy

**"One Abstraction Away"**

The LLM (Brain Layer) plans and coordinates. Traditional AI (Script Layer) executes in real-time. The Minecraft API (Physical Layer) handles world interaction.

Each layer operates at its natural timescale:
- Brain: Every 30-60 seconds (planning)
- Script: Every tick (20 TPS) (execution)
- Physical: Every tick (game interaction)

### Key Patterns

**Most Frequently Used:**
1. **Strategy Pattern** - LLM providers, pathfinding
2. **State Machine** - Agent lifecycle, actions
3. **Async Request-Response** - LLM calls, HTTP
4. **Plugin Architecture** - Actions, skills
5. **Builder Pattern** - Configuration
6. **Repository Pattern** - Persistence
7. **Publish-Subscribe** - Events
8. **Blackboard** - Shared knowledge

**When in doubt, check [PATTERN_LANGUAGE.md](PATTERN_LANGUAGE.md).**

### Common Pitfalls

**Avoid these:**
1. **Getting lost in details** - Start with architecture, not implementation
2. **Premature optimization** - Measure first, optimize second
3. **Pattern blindness** - Actively look for patterns
4. **Overconfidence** - Test your hypotheses
5. **Analysis paralysis** - Set time limits, make progress

**For more, see [META_COGNITION.md](META_COGNITION.md) section 7.**
