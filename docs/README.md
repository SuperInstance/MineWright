## Contributing Guidelines

### Adding New Documentation

When adding new documentation to the `docs/` folder:

1. **Choose the Right Template**
   - Use **Design Document** for feature specifications
   - Use **Implementation Guide** for code changes
   - Use **Analysis** for research and recommendations
   - Use **Quick Reference** for cheat sheets

2. **Follow Naming Conventions**
   ```
   FEATURE_NAME.md                    - Main design doc
   FEATURE_NAME_QUICK_START.md        - Quick start guide
   FEATURE_NAME_ARCHITECTURE.md       - Architecture diagrams
   FEATURE_NAME_TEST_PLAN.md          - Test strategy
   FEATURE_NAME_INDEX.md              - Navigation index (for large features)
   ```

3. **Include Required Sections**
   ```markdown
   # Title

   **Status:** [Status from legend]
   **Date:** YYYY-MM-DD
   **Version:** X.Y.Z

   ## Executive Summary
   [2-3 sentence overview]

   ## Table of Contents
   [Auto-generated]

   ## Document Body
   [Content]

   ## Status
   [Current implementation status]

   ---
   **Document Version:** X.Y
   **Last Updated:** YYYY-MM-DD
   ```

4. **Update This Index**
   - Add your document to the appropriate category
   - Update document counts
   - Add quick links if it's a major feature

### Documentation Review Process

Before marking a document as "Ready for Implementation":

- [ ] Design is complete and reviewed
- [ ] All edge cases are considered
- [ ] Integration points are documented
- [ ] Testing strategy is defined
- [ ] Performance implications are analyzed
- [ ] This index is updated
